## What's new in <em>v2</em>

Same CLI, same output filenames, same report format — with a few capabilities the Perl version did not have.

<div class="tg-news not-content">
  <div class="tg-news-item">
    <div class="num">01</div>
    <h3>Single binary</h3>
    <p>No Python, Perl, Cutadapt, Java, igzip, or pigz dependencies. Install via <code>cargo install trim-galore</code>, from bioconda, or pull a Docker image.</p>
  </div>
  <div class="tg-news-item">
    <div class="num">02</div>
    <h3>Single-pass paired-end</h3>
    <p>Both reads are processed together in one pass. No temp files, no separate adapter / quality / length validation passes.</p>
  </div>
  <div class="tg-news-item">
    <div class="num">03</div>
    <h3>Poly-G auto-detect</h3>
    <p>Trims no-signal G-runs from 2-colour instruments. Auto-enabled when the data looks 2-colour. Independent of <code>--nextseq</code> quality trimming.</p>
  </div>
  <div class="tg-news-item">
    <div class="num">04</div>
    <h3>Smarter adapter detection</h3>
    <p>Adapter auto-detection now runs per file (or file-pair) instead of once on the first input. BGI/DNBSEQ joins the probe set alongside Illumina, Nextera, and smallRNA.</p>
  </div>
  <div class="tg-news-item">
    <div class="num">05</div>
    <h3>Built-in FastQC</h3>
    <p>FastQC reports run in-process via the bundled <code>fastqc-rust</code> library. No Java, no external <code>fastqc</code> binary. Pass <code>--fastqc</code> and it just works.</p>
  </div>
</div>

## One command

Defaults work for most Illumina libraries. Auto-detected adapter, Phred quality trimming, paired-end validation.

```bash
trim_galore --cores 8 sample.fastq.gz                                  # single-end
trim_galore --cores 8 --paired sample_R1.fastq.gz sample_R2.fastq.gz   # paired-end
```
