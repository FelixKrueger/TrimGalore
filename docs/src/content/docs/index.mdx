## What's new in <em>v2</em>



Same CLI, same output filenames, same report format. A few capabilities the Perl version did not have.

<div class="tg-news not-content">
  <div class="tg-news-item">
    <div class="num">01</div>
    <h3>Single binary</h3>
    <p>No Python, no Cutadapt, no pigz. <code>cargo install trim-galore</code>, install from bioconda, or pull a Docker image. 5 MB resident memory single-threaded.</p>
  </div>
  <div class="tg-news-item">
    <div class="num">02</div>
    <h3>Single-pass paired-end</h3>
    <p>Both reads are processed together in one pass. No temp files, no three-pass Cutadapt orchestration.</p>
  </div>
  <div class="tg-news-item">
    <div class="num">03</div>
    <h3>Poly-G auto-detect</h3>
    <p>Trims no-signal G-runs from 2-colour instruments. Auto-enabled when the data looks 2-colour. Independent of <code>--nextseq</code> quality trimming.</p>
  </div>
  <div class="tg-news-item">
    <div class="num">04</div>
    <h3>Multi-adapter input</h3>
    <p>Repeatable <code>-a</code> / <code>-a2</code>, FASTA file input (<code>file:adapters.fa</code>), and <code>A&#123;N&#125;</code> brace expansion. Adapter auto-detection runs per pair.</p>
  </div>
</div>

## One command

Defaults work for most Illumina libraries. Auto-detected adapter, Phred quality trimming, paired-end validation.

```bash
# Paired-end with parallel processing
trim_galore --cores 8 --paired sample_R1.fastq.gz sample_R2.fastq.gz
```
