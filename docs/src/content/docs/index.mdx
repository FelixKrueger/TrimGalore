## What's new in v2

Trim Galore v2 is a Rust rewrite of the Perl wrapper. Same CLI, same output filenames, same report format. The binary has no external dependencies, and it adds a few capabilities the Perl version did not have.

<CardGrid>
  <Card title="Single binary" icon="rocket">
    No Python, no Cutadapt, no pigz. `cargo install trim-galore`, install from bioconda, or pull a Docker image. 5 MB RAM single-threaded.
  </Card>
  <Card title="Single-pass paired-end" icon="random">
    Both reads are processed together in one pass. No temp files, no three-pass Cutadapt orchestration.
  </Card>
  <Card title="Poly-G auto-detect" icon="seti:graphql">
    Trims no-signal G-runs from 2-colour instruments (NovaSeq, NextSeq, NovaSeq X). Auto-enabled when the data looks 2-colour. Independent of `--nextseq` quality trimming.
  </Card>
  <Card title="Multi-adapter input" icon="setting">
    Repeatable `-a` / `-a2`, FASTA file input (`file:adapters.fa`), and `A{N}` brace expansion. Adapter auto-detection runs per pair.
  </Card>
  <Card title="Faster on the same cores" icon="bars">
    1.9× faster than v0.6.x at the nf-core 8-core allocation, using 2.6× less CPU time. 4.4× faster at `--cores 24`.
  </Card>
  <Card title="MultiQC compatible" icon="document">
    The text trimming report parses cleanly in existing MultiQC dashboards. v2 also writes a structured JSON report alongside it.
  </Card>
</CardGrid>
